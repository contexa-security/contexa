// THE SOLUTION

맥락으로 판단합니다

자격증명이 아니라 행동의 진정성을 검증합니다. 공격자가 열쇠를 훔쳐도, 행동 환경 전체를 복제하는 것은 불가능합니다.

// 기존 보안 — 자격증명 검증
✓ ID / PW 가 맞는가?
✓ 토큰이 유효한가?
✓ 권한이 있는가?
→ 세 가지 통과 = 무조건 허용
퇴직자도, 공격자도 들어온다
VS
// Contexa — 행동의 진정성 검증
✓ 평소 접속 위치와 같은가?
✓ 접속 시간대가 정상인가?
✓ 디바이스 / OS / Agent 가 같은가?
✓ 요청 흐름이 업무 패턴과 일치하는가?
→ 모두 일치해야만 허용
행동 환경 전체를 복제하는 것은 불가능
// HOW IT WORKS

3-Layer 행동 기반 분석

모든 요청을 LLM으로 처리하지 않습니다. @Protectable 민감 리소스에만 정밀 분석이 작동합니다.

01
Behavioral Baseline
사용자의 행동 지문을 수집합니다. 위치 · 시간 · 디바이스 · OS · Agent · 요청 흐름. 리소스 정보를 제외한 순수한 행동 패턴만 학습합니다.
처리시간: < 10ms
02
RAG 유사도 분석
과거 허용된 정상 패턴과 현재 요청을 비교합니다. Deny 패턴 저장 금지로 RAG 오염 공격을 원천 차단합니다.
처리시간: < 100ms
03
LLM 최종 판단 — @Protectable 전용
민감 리소스 접근 시에만 의미와 맥락으로 판단합니다. 규칙 없이 의도를 읽습니다. 새로운 공격 패턴도 별도 업데이트 없이 탐지합니다.
처리시간: 비동기 백그라운드 처리
// QUICK START

의존성 하나면 됩니다

기존 Spring Security 코드를 변경하지 않아도 됩니다. Contexa는 Spring Security 위에서 작동합니다.

build.gradle
// 1. 의존성 추가
implementation 'io.contexa:spring-boot-starter-contexa:1.0.0'
UserController.java
// 2. 민감 리소스에 어노테이션 선언

@Protectable
@GetMapping("/api/customers")
public ResponseEntity<?> getCustomers() {
    // 기존 코드 그대로
    return customerService.findAll();
}
application.yml
# 3. 모델 설정 (고객이 선택)

contexa:
  ai:
    model: gpt-4o  # or claude-3-5, ollama...
  baseline:
    learning-period: 7d
    allow-cache-ttl: 30m
STEP 01
의존성 추가
build.gradle 또는 pom.xml에 한 줄 추가합니다.
STEP 02
@Protectable 선언
민감 API에 어노테이션만 붙이면 자동으로 보호됩니다.
STEP 03
모델 선택
고객 환경에 맞는 LLM을 yml에서 선택합니다.
// TECHNICAL MOAT

설계로 만든 해자

시간이 지날수록 따라잡기 어려운 구조적 차별화입니다.

🧠
규칙 완전 제거
규칙은 공격자가 학습하고 우회합니다. Contexa는 규칙 없이 의미로 판단합니다.
🛡️
RAG 오염 방지
Allow 패턴만 학습합니다. Deny 저장 금지로 RAG를 역이용하는 공격을 차단합니다.
🔒
리소스 레벨 제외
리소스 정보가 RAG에 없어 역공학이 불가능합니다. 리소스 추가 시 재학습도 불필요합니다.
⚡
Spring Native 통합
Spring Security 심장부에서 작동합니다. Okta, Auth0는 외부에서 감싸는 구조입니다.
🎯
최소 침습 통합
기존 코드 변경 없이 어노테이션 하나로 작동합니다. 전면 교체가 필요 없습니다.
🔓
오픈소스 Core
Spring 생태계 표준으로 자리잡습니다. 5,000명 현직 개발자 채널로 즉시 확산됩니다.
