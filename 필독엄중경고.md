## 프로젝트 정보

**프로젝트명**: Contexa
**타입**: Spring Boot 3.x + Multi-Factor Authentication Platform
**주요 모듈**:
- contexa-core: 핵심 비즈니스 로직
- contexa-identity: 인증/인가 시스템
- contexa-iam: 접근 제어 (XACML)

**개발 언어**: Java 21, Spring Boot 3.4+, Spring Security 6.4+

## 코드 작성 컨벤션

### Java 코드 주석
```java
/**
 * MFA 요소 처리 성공 핸들러
 *
 * <p>
 * WebAuthn, OTT 등 다양한 인증 방식의 성공 처리를 담당합니다.
 * State Machine과 통합되어 OAuth2 토큰을 자동 발급합니다.
 * </p>
 *
 * @param authentication 인증 객체
 * @return 토큰 발급 결과
 */
public TokenResponse handleSuccess(Authentication authentication) {
    // 구현...
}
```

### 클래스 참조 규칙
**중요**: 코드와 문서에서 클래스를 참조할 때는 **클래스명만** 사용합니다.

**올바른 예시**:
```java
// ✅ 좋음
MfaFactorProcessingSuccessHandler handler = ...;
log.info("핸들러 교체: {}", MfaFactorProcessingSuccessHandler.class.getSimpleName());

// ✅ 좋음 (문서)
`MfaFactorProcessingSuccessHandler`가 토큰을 발급합니다.
```

**잘못된 예시**:
```java
// 나쁨 - 풀네임 사용 금지
io.contexa.contexaidentity.security.handler.MfaFactorProcessingSuccessHandler handler = ...;

// 나쁨 (문서)
`io.contexa.contexaidentity.security.handler.MfaFactorProcessingSuccessHandler` 사용
```

**예외 상황** (풀네임 사용 가능):
- import 문
- 패키지 구조 설명 문서
- 동일한 클래스명이 여러 패키지에 있어 명확화가 필요한 경우
