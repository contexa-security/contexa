🎯 핵심 원칙
1. 점진적 전환 (병행 운영)
// 전환 중: ComponentScan + AutoConfiguration 병행
@SpringBootApplication(
    scanBasePackages = {
        "io.contexa.contexaidentity",
        "io.contexa.contexacore.autonomous",    // 아직 AutoConfiguration 안된 것만
        // "io.contexa.contexacore.hcad",       // ← AutoConfiguration 완료
        // "io.contexa.contexacore.std"         // ← AutoConfiguration 완료
    }
)
2. 의존성 순서 엄격 관리
HCAD, Autonomous는 의존성 그래프 명확히 파악 후 순서대로 Bean 등록
순환 참조 발견 시 @Lazy 또는 인터페이스 분리
3. @ConditionalOnProperty 적극 활용
@ConditionalOnProperty(prefix = "contexa.hcad", name = "enabled", havingValue = "true", matchIfMissing = true)
4. Enterprise 패턴 유지
EnterpriseBeanConfiguration의 함수형 인터페이스 export 패턴 유지
Helper 클래스들 @Component 유지 (이미 잘 작동)
5. 테스트 주도
각 AutoConfiguration마다 단위 테스트 작성
통합 테스트로 전체 동작 검증
🚨 위험 요소 및 대응 방안
위험 요소 확률  영향  대응 방안
HCAD 순환 참조  높음  높음  @Lazy 주입, 인터페이스 분리, 충분한 분석 시간
Autonomous 복잡도  높음  중간  단계별 전환, 철저한 테스트
