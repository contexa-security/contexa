테스트 시나리오 가이드

시나리오 1: 정상 사용자 + 공개 데이터

정상 사용자 시나리오 선택 후 공개 데이터 테스트. 1차/2차 모두 성공합니다.

시나리오 2: 정상 사용자 + 민감 데이터

1차 요청에서 분석 트리거, 3초 대기 후 2차 요청에서 ALLOW로 접근 허용됩니다.

시나리오 3: 계정 탈취 + 민감 데이터

LLM이 비정상 컨텍스트를 감지, 2차 요청에서 BLOCK/CHALLENGE로 차단됩니다.

시나리오 4: 봇 공격 + 일반 데이터

curl User-Agent를 LLM이 봇으로 판별, 2차 요청에서 BLOCK으로 차단됩니다.

시나리오 5: 초기화 후 REQUIRED 테스트

분석 결과 초기화 후 민감 데이터 접근 시 1차 요청에서 분석 대기 타임아웃 발생.

시나리오 6: 동일 계정 + 다른 컨텍스트

같은 사용자라도 IP/UA가 다르면 LLM이 다르게 판단합니다.
