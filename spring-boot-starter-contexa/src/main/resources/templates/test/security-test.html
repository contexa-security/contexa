LLM Action 설정 (테스트용)

실제 LLM 분석 결과를 시뮬레이션합니다. 설정된 Action은 Redis에 저장되어 SpEL 표현식 평가에 사용됩니다.

ALLOW: 정상 사용자로 판단되어 접근을 허용합니다.

보안 테스트 실행

각 버튼을 클릭하여 @Protectable 메서드를 호출합니다. 현재 설정된 Action과 정책 조건에 따라 허용 또는 차단됩니다.

공개 데이터 조회

NOT_REQUIRED

인증된 사용자만 허용 (Action 무관)

isAuthenticated()

일반 데이터 조회

PREFERRED

ALLOW 또는 MONITOR Action 필요

#trust.hasActionIn('ALLOW', 'MONITOR')

민감 데이터 조회

REQUIRED

분석 완료 + ALLOW/MONITOR 필수

#trust.requiresAnalysisWithAction('ALLOW', 'MONITOR')

중요 데이터 조회

STRICT

ADMIN 권한 + ALLOW만 허용

hasRole('ADMIN') and #trust.requiresAnalysisWithAction('ALLOW')

대량 데이터 조회

Runtime Interception

BLOCK 아니면 허용 (기본 MONITOR)

#trust.hasActionOrDefault('MONITOR', 'ALLOW', 'MONITOR')

테스트 시나리오 가이드

시나리오 1: NOT_REQUIRED 테스트

어떤 Action이든 공개 데이터는 접근 가능합니다.

시나리오 2: PREFERRED + ALLOW

Action을 ALLOW로 설정하고 일반 데이터 조회 시 성공합니다.

시나리오 3: PREFERRED + BLOCK

Action을 BLOCK으로 설정하고 일반 데이터 조회 시 차단됩니다.

시나리오 4: REQUIRED + PENDING_ANALYSIS

초기화 버튼 클릭 후 민감 데이터 조회 시 분석 대기 후 타임아웃으로 차단됩니다.

시나리오 5: STRICT + ALLOW (ADMIN)

ADMIN 계정으로 로그인, Action을 ALLOW로 설정하면 중요 데이터 접근 가능합니다.

시나리오 6: STRICT + MONITOR

ADMIN 계정이어도 MONITOR Action은 중요 데이터 접근이 차단됩니다.
