contexa-common

Annotations

Core annotations defined in contexa-common that enable AI security activation and method-level protection across the Contexa platform.

@EnableAISecurity

Activates the Contexa AI-Native Zero Trust security infrastructure. Place this annotation on a @SpringBootApplication class to bootstrap the full security configurer mechanism.

Java 
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Import(AiSecurityImportSelector.class)
public @interface EnableAISecurity {
}

Attributes

This annotation has no configurable attributes. All configuration is driven by application.yml properties.

Internal Mechanism

The @Import(AiSecurityImportSelector.class) meta-annotation triggers the following:

  1. AiSecurityImportSelector checks for an existing PlatformConfig bean
  2. If no PlatformConfig exists, it creates a default one using IdentityDslRegistry
  3. The PlatformConfig activates the full Zero Trust configurer mechanism
  4. Security filter chains, XACML components, and AI services are initialized

Prerequisites

  • spring-boot-starter-security must be on the classpath
  • Legacy systems integrating with Contexa must declare this dependency explicitly

Usage

Java 
@SpringBootApplication
@EnableAISecurity
public class LegacyApplication {
    public static void main(String[] args) {
        SpringApplication.run(LegacyApplication.class, args);
    }
}

@Protectable

Marks a method for AI-driven authorization enforcement. When a protected method is invoked, the XACML PEP intercepts the call, evaluates security rules via SpEL expressions, and optionally performs synchronous AI risk analysis.

Java 
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Protectable {
    String ownerField() default "";
    boolean sync() default false;
}

Attributes

AttributeTypeDefaultDescription
ownerField String "" The field name on the request body or path variable that identifies the resource owner. When empty, the entire resource is protected without owner-based scoping. When set, the authorization engine can enforce owner-only access patterns.
sync boolean false Enables synchronous AI analysis before the method proceeds. When true, the SynchronousProtectableDecisionService performs real-time risk assessment and returns a ZeroTrustAction decision (ALLOW, BLOCK, CHALLENGE, ESCALATE, or PENDING_ANALYSIS).

Usage Examples

Java 
// Basic protection - asynchronous evaluation, no owner filtering
@Protectable
@PostMapping("/api/users/{id}/disable")
public void disableUser(@PathVariable Long id) { ... }

// Owner-scoped with synchronous AI analysis
@Protectable(ownerField = "userId", sync = true)
@PutMapping("/api/accounts/{id}")
public Account updateAccount(@PathVariable Long id,
                             @RequestBody AccountDto dto) { ... }

For detailed processing pipeline documentation, see the @Protectable (IAM) reference page.
