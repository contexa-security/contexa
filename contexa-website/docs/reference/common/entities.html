JPA entity model for the Contexa platform covering core identity entities, relationship mappings, behavior analytics entities, and business resource entities.

Core Entities

Users

The primary user entity. Uses username as the unique login identifier.

ColumnTypeConstraintsDescription
idLongPK, auto-generatedUser ID
usernameStringunique, not nullLogin identifier
passwordStringnot nullEncoded password
nameStringnot nullDisplay name
mfaEnabledbooleannot nullMFA activation status
preferredMfaFactorStringPreferred MFA factor type
lastUsedMfaFactorStringLast used MFA factor (fallback for preferred)
createdAtLocalDateTimeAccount creation timestamp
updatedAtLocalDateTimeLast update timestamp
lastMfaUsedAtLocalDateTimeLast MFA usage timestamp

Transient methods getRoleNames() and getPermissionNames() traverse the full relationship chain (Users → UserGroup → Group → GroupRole → Role → RolePermission → Permission) to collect aggregated role and permission names.

Role

Represents an authorization role. Table: ROLE.

ColumnTypeConstraintsDescription
id (role_id)LongPK, auto-generatedRole ID
roleName (role_name)Stringunique, not nullRole name (e.g., ADMIN, USER)
roleDesc (role_desc)StringHuman-readable description
isExpression (is_expression)StringExpression-based role flag

Permission

Represents a fine-grained permission. Table: PERMISSION. Has a 1:1 relationship with ManagedResource.

ColumnTypeConstraintsDescription
id (permission_id)LongPK, auto-generatedPermission ID
name (permission_name)Stringunique, not nullPermission name
friendlyNameStringDisplay name
descriptionString(1024)Purpose description
targetTypeStringTarget type (e.g., METHOD, URL)
actionTypeStringAction type (e.g., READ, WRITE)
conditionExpressionString(2048)SpEL condition expression
managedResourceManagedResourceFK, uniqueLinked managed resource

Group

Logical grouping of users. Table: APP_GROUP.

ColumnTypeConstraintsDescription
id (group_id)LongPK, auto-generatedGroup ID
name (group_name)Stringunique, not nullGroup name
descriptionStringGroup description

ManagedResource

Represents a protectable resource discovered by resource scanners. Table: MANAGED_RESOURCE. See Resource Scanner for details.

ColumnTypeDescription
idLongPK, auto-generated
resourceIdentifierString(512)URL pattern or method signature
resourceTypeEnum: URL, METHODResource type
httpMethodEnum: GET, POST, PUT, DELETE, PATCH, ANYHTTP method (URL type only)
friendlyNameStringHuman-readable name
descriptionString(1024)Resource description
serviceOwnerStringOwning class name
statusEnum: NEEDS_DEFINITION, PERMISSION_CREATED, POLICY_CONNECTED, EXCLUDEDLifecycle status

Relationship Entities

Join entities that model the many-to-many relationships between core entities. All use composite keys and cascade delete with orphan removal.

EntityLeft SideRight SideDescription
UserGroup Users (user) Group (group) Assigns users to groups
GroupRole Group (group) Role (role) Assigns roles to groups
RolePermission Role (role) Permission (permission) Assigns permissions to roles

The authorization chain traversal is: Users → UserGroup → Group → GroupRole → Role → RolePermission → Permission → ManagedResource.

Behavior Entities

Entities for AI-driven behavioral analysis, stored in the io.contexa.contexacommon.entity.behavior package.

UserBehaviorProfile

Stores aggregated behavioral profiles for each user. Table: user_behavior_profiles.

ColumnTypeDescription
idLongPK, auto-generated
userIdStringUser identifier
profileTypeString(50)Profile type classification
vectorClusterIdStringBehavioral vector cluster assignment
clusterCentroidVectorTEXTCentroid vector of the assigned cluster
clusterSizeIntegerNumber of data points in the cluster
normalRangeMetadataJSONNormal behavioral range metadata
commonActivitiesJSONFrequently performed activities
commonIpRangesJSONCommonly used IP address ranges

BehaviorAnomalyEvent

Records individual anomaly events detected by the HCAD (Hybrid Contextual Anomaly Detection) engine. Table: behavior_anomaly_events.

ColumnTypeDescription
idLongPK, auto-generated
userIdStringUser who triggered the anomaly
eventTimestampLocalDateTimeWhen the anomaly occurred
activityString(500)Activity that triggered detection
remoteIpString(45)Source IP address
anomalyScoredoubleAnomaly score (0.0-1.0)
riskLevelString(20)Computed risk level
anomalyFactorsJSONContributing factors to the anomaly score
aiAnalysisIdStringReference to the AI analysis record

BehaviorBasedPermission

Dynamic permissions granted or restricted based on behavioral analysis.

BehaviorRealtimeCache

Real-time behavioral data cache for low-latency anomaly detection during authorization checks.

Business Entities

Entities for business-level resource and action modeling, stored in io.contexa.contexacommon.entity.business.

EntityDescription
BusinessResource Represents a business-level resource (e.g., "Customer Data", "Financial Reports") with resource type classification and sensitivity metadata.
BusinessAction Represents a business-level action (e.g., "View", "Export", "Approve") that can be performed on business resources.
BusinessResourceAction Maps which actions are allowed on which resources. Acts as the join entity between BusinessResource and BusinessAction.

Business entities are used by the DatabaseAttributePIP to enrich authorization context with resource sensitivity levels and by the BusinessPolicyService to translate business rules into technical policies.
