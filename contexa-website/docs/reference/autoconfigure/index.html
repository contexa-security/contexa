Auto-Configuration Catalog

Complete catalog of 45 auto-configuration classes in the contexa-autoconfigure module, organized by category. Each class uses Spring Boot's conditional configuration to activate based on classpath presence, property values, and deployment mode.

Overview

The contexa-autoconfigure module follows Spring Boot's auto-configuration conventions. Configuration classes are registered via META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports and use @Conditional* annotations to activate only when their prerequisites are met.

Key conditional patterns used throughout:

  • @ConditionalOnClass — Activates when a specific class is on the classpath
  • @ConditionalOnMissingBean — Activates only if the bean is not already defined by user configuration
  • @ConditionalOnProperty — Activates based on application property values (e.g., standalone vs. distributed mode)
  • @ConditionalOnBean — Activates when a dependent bean exists in the context
