Get Started View on GitHub
1 min Setup Add dependency, add two annotations. Done.
Java 
@SpringBootApplication
@EnableAISecurity
public class MyApplication {

    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}

@Service
public class OrderService {

    @Protectable
    public Order getOrder(Long orderId) {
        return orderRepository.findById(orderId);
    }
}

Two annotations are all you need.

@EnableAISecurity
Activates AI-driven Zero Trust. One annotation on your main class, full protection.
@Protectable
Owner-based access control on any method. AI verifies every call against the user's baseline.

Features

Built for Modern Security

A comprehensive security platform that unifies identity, authorization, and AI-driven threat analysis in a single Spring-native integration.

Identity DSL

Declarative authentication flow configuration through IdentityDslRegistry. Supports Form Login, REST, One-Time Token, Passkey, and MFA — all composable via a fluent DSL API.

Dynamic Authorization

XACML-based policy evaluation with the @Protectable annotation. Protect any service method with owner-based access control and synchronous enforcement options.

AI Security Engine

Hybrid Cold/Hot-path Anomaly Detection (HCAD). The Hot Path handles 98% of requests in 5–30ms with rule-based evaluation. The Cold Path escalates the remaining 2% to LLM analysis.

Zero Trust Actions

Five graduated response actions for fine-grained access control.

ALLOW BLOCK CHALLENGE ESCALATE PENDING_ANALYSIS

Shadow Mode

Safe 4-stage migration path for production deployments. Progress through Shadow, Advisory, Hybrid, and Autonomous modes — each stage increasing AI decision authority while preserving observability.

Streaming & RAG

Server-Sent Events (SSE) streaming for real-time AI responses. PgVector-backed Retrieval-Augmented Generation for context-aware threat analysis with cosine distance similarity search.

Infrastructure

Operating Modes

Choose the deployment topology that fits your environment. Switch between modes with a single configuration property.

Standalone

All Contexa components run within a single Spring Boot application. Uses in-memory caching and local event handling. Ideal for development, testing, and single-instance deployments.

  • In-memory cache with Caffeine
  • Local event publishing
  • Zero external dependencies
  • Default mode (no configuration needed)

Distributed

Contexa leverages Redis for shared caching, distributed sessions, and cross-instance event propagation. Designed for clustered and high-availability production environments.

  • Redis-backed shared cache
  • Cross-instance event propagation
  • Horizontal scaling support
  • Session sharing across nodes
YAML 
# Standalone (default — no external dependencies)
contexa:
  infrastructure:
    mode: standalone

# Distributed (requires Redis)
contexa:
  infrastructure:
    mode: distributed

spring:
  data:
    redis:
      host: localhost
      port: 6379
