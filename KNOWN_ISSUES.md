# 알려진 이슈 및 제한사항 (Known Issues)

**버전**: 0.1.0-SNAPSHOT
**상태**: POC (Proof of Concept)
**최종 업데이트**: 2025-11-18

---

## ⚠️ 중요 공지

**이 프로젝트는 현재 POC 단계입니다.** 프로덕션 환경에서 사용하기 전에 반드시 아래 이슈들을 검토하고 필요한 수정을 완료해야 합니다.

---

## 1. Critical 이슈 (즉시 수정 필요)

### 1.1 보안 관련
- **상태**: 🔴 미해결
- **설명**: 일부 엔드포인트에 대한 권한 검증이 불완전할 수 있음
- **영향**: 권한 없는 사용자가 특정 API에 접근할 수 있는 잠재적 위험
- **해결 방안**: 모든 엔드포인트에 대한 포괄적인 권한 검증 구현 필요

### 1.2 데이터베이스 트랜잭션
- **상태**: 🔴 미해결
- **설명**: 복잡한 비즈니스 로직에서 트랜잭션 경계 설정이 불명확한 부분 존재
- **영향**: 데이터 일관성 문제 발생 가능
- **해결 방안**: @Transactional 적용 범위 재검토 및 격리 수준 명시

---

## 2. High 이슈 (우선 수정 권장)

### 2.1 성능 최적화
- **상태**: 🟡 부분 해결
- **설명**: N+1 쿼리 문제가 일부 엔티티 관계에서 발생
- **영향**: 대량 데이터 처리 시 성능 저하
- **해결 방안**: @EntityGraph 또는 fetch join 적용 필요
- **관련 파일**:
  - `Users.java` - userGroups 관계
  - `Group.java` - groupRoles 관계

### 2.2 캐싱 전략
- **상태**: 🔴 미해결
- **설명**: Redis 캐시 설정은 되어 있으나 TTL 및 eviction 정책이 미정의
- **영향**: 메모리 부족 또는 stale data 반환 가능
- **해결 방안**: 캐시 전략 수립 및 TTL 설정 필요

### 2.3 비동기 처리
- **상태**: 🔴 미해결
- **설명**: 일부 무거운 작업(AI 호출, 대량 데이터 처리)이 동기로 실행됨
- **영향**: 응답 시간 지연 및 사용자 경험 저하
- **해결 방안**: CompletableFuture 또는 @Async 적용 필요

---

## 3. Medium 이슈 (일반 수정 필요)

### 3.1 테스트 커버리지
- **상태**: 🟡 부분 해결
- **설명**: 단위 테스트 커버리지 약 30-40% 추정
- **영향**: 코드 변경 시 회귀 버그 발견 어려움
- **해결 방안**: 핵심 비즈니스 로직에 대한 테스트 작성 우선

### 3.2 예외 처리
- **상태**: 🟡 부분 해결
- **설명**: 일부 메서드에서 예외를 포괄적으로 catch하여 원인 파악 어려움
- **영향**: 디버깅 어려움 및 로그 품질 저하
- **해결 방안**: 구체적인 예외 타입 정의 및 처리 필요

### 3.3 로깅 일관성
- **상태**: 🔴 미해결
- **설명**: 로그 레벨 및 메시지 형식이 일관되지 않음
- **영향**: 운영 중 문제 추적 어려움
- **해결 방안**: 로깅 가이드라인 수립 및 적용

### 3.4 API 문서화
- **상태**: 🔴 미해결
- **설명**: Swagger/OpenAPI 문서가 불완전
- **영향**: API 사용자가 엔드포인트 이해 어려움
- **해결 방안**: 모든 엔드포인트에 @Operation, @Schema 적용

---

## 4. Low 이슈 (개선 사항)

### 4.1 코드 중복
- **상태**: 🟡 부분 해결
- **설명**: DTO 변환 로직, 유효성 검증 로직 등에서 코드 중복 발견
- **영향**: 유지보수성 저하
- **해결 방안**: 공통 유틸리티 클래스 또는 AOP 활용

### 4.2 설정 외부화
- **상태**: 🟡 부분 해결
- **설명**: 일부 설정값이 코드에 하드코딩됨
- **영향**: 환경별 배포 어려움
- **해결 방안**: application.yml 또는 환경 변수로 외부화

### 4.3 네이밍 일관성
- **상태**: 🟡 부분 해결
- **설명**: 일부 클래스/메서드 네이밍이 관례를 따르지 않음
- **영향**: 코드 가독성 저하
- **해결 방안**: 코드 리뷰를 통한 점진적 개선

## 5. 기술 부채 (Technical Debt)

### 5.1 Deprecated API 사용
- **상태**: 🟡 확인됨
- **설명**: RestTemplateBuilder.setConnectTimeout(Duration) 등 deprecated 메서드 사용
- **영향**: 향후 Spring Boot 버전 업그레이드 시 호환성 문제
- **해결 방안**: Spring Boot 공식 문서 확인 후 최신 API로 마이그레이션

### 5.2 레거시 코드
- **상태**: 🔴 미해결
- **설명**: 일부 초기 구현 코드가 리팩토링 없이 남아 있음
- **영향**: 코드 복잡도 증가 및 유지보수 어려움
- **해결 방안**: 점진적 리팩토링 계획 수립
