## 절대 규칙 (Absolute Rules) - Thinking 과정: 반드시 한국어 사용

### 규칙 1: 필독.md 우선 읽기
**매 작업, compacting 후, 세션 시작 시 필독.md를 반드시 먼저 읽는다**

- 모든 작업 전 필독.md 확인 필수
- 규칙 준수 여부 점검
- 위반 시 즉시 작업 중단

**강제 사항**: 세션 시작 시 자동으로 필독.md 읽기

---

### 규칙 2: 클래스명만 사용 (풀네임 절대 금지)
**코드와 문서에서 클래스를 참조할 때는 클래스명만 사용한다**

#### 금지 패턴
```java
// ❌ 절대 금지
java.util.Set<String> getPendingApprovalIds();
org.springframework.stereotype.Component
io.contexa.contexacore.domain.Entity
```

#### 허용 패턴
```java
// ✅ 올바른 방법
import java.util.Set;
Set<String> getPendingApprovalIds();

import org.springframework.stereotype.Component;
@Component

import io.contexa.contexacore.domain.Entity;
Entity entity;
```

**강제 사항**:
- 타입 선언 시 클래스명만 사용
- 파일 상단에 import 문 필수 추가
- 풀네임 사용 시 즉시 중단 및 수정

---

### 규칙 3: 100% 한국어 소통
**설명, 요약, 대화: 100% 한국어**

- 모든 사용자 대화는 한국어로만 작성
- 응답, 설명, 요약 모두 한국어
- 기술 용어 영어 병기는 허용 (예: "필터 체인(Filter Chain)")

**강제 사항**: 영어 응답 절대 금지

---

### 규칙 4: Thinking 과정 한국어
**Thinking 과정: 반드시 한국어 사용**

- 내부 사고 과정도 한국어로만 작성
- 문제 분석, 계획 수립 모두 한국어
- 예외 없음

**강제 사항**: 영어 thinking 절대 금지

**기술 용어: 영어 병기 허용**

#### 허용 패턴

### 규칙 6: 이모지 절대 금지
**코드, 주석, 로그 등: 이모지 절대 사용 불가**

#### 금지 대상
```java
// ❌ 절대 금지
public class Service {
    // ✅ 성공
    log.info("처리 완료 ✅");  // 금지

    /**
     * 🔍 검색 메서드  // 금지
     */
}
```

#### 허용 패턴
```java
// ✅ 올바른 방법
public class Service {
    // 성공
    log.info("처리 완료");

    /**
     * 검색 메서드
     */
}
```

**강제 사항**:
- Java 파일에 이모지 절대 금지
- 주석에도 이모지 금지
- 로그 메시지에도 이모지 금지
- Markdown 문서는 허용

### 규칙 7: 참고 파일 정확히 읽기
**사용자가 참고하라고 한 파일을 즉시 읽고 패턴을 정확히 파악**

- 대충 분석 절대 금지
- 전체 파일 읽기 필수
- 패턴 정확히 이해 후 적용
- 추측하지 말고 확인

**강제 사항**:
- Read tool로 전체 파일 읽기
- 패턴 분석 후 정확히 적용
- 불확실하면 다시 읽기
